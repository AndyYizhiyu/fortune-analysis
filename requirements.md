# 📝 个性化命理提示词生成器 - 完整需求文档

---

## 1. 项目概述

### 1.1 项目背景

本项目是一个基于 DeepSeek API（默认）和 Kimi API 的**命理分析提示词生成工具**。用户填写出生信息、出生地、**可选**补充名字中的五行元素，并可补充性别、星座、MBTI 和当前关注方向；后端将用户事实与**调用大模型用的系统指令**一并提交给 DeepSeek/Kimi，由该模型**动态生成**一段与 §2.2 结构一致的、**可复制到其它大模型对话中的「下游用提示词」全文**（**非**仅靠后端本地静态模板拼接作为最终交付物）。**本服务接入的 DeepSeek/Kimi 在本接口的回复中不承担对用户本人的八字排盘、十神、流年断语或具体命理建议**；这些由用户在下游对话中粘贴生成结果后，由**下游大模型**依据提示词中的用户信息与任务要求自行分析完成。

### 1.2 目标用户

- 想为自己或他人生成命理分析提示词的普通用户
- 希望结合八字、星座、MBTI 获得工作、感情、学业等建议的用户

### 1.3 核心价值

- 由 DeepSeek/Kimi **只生成「给下游模型用的提示词」**，边界清晰、可审计
- 用户可在任意下游大模型中粘贴该提示词，再获得命理类分析（本需求不规定下游模型）
- 降低用户手写长提示词的成本，并统一语气与结构约束（写入下游提示词正文中）

## 2. 功能需求

### 2.1 输入模块

| 字段名 | 类型 | 必填 | 约束 | 说明 |
|--------|------|------|------|------|
| 出生年月日 | 年、月、日三个下拉框 | ✅ | 公历年范围为 1945 年到当前年份；月范围 1-12；日根据所选年月动态生成；组合日期必须真实存在且不能超过当前日期 | 年月日放在同一行；使用统一美化的下拉框样式；组合为 `YYYY-MM-DD` 后自动计算星座 |
| 出生时间 | 文本框 | ❌ | 可填写模糊时间，如“凌晨一点左右”；建议具体到分钟，如 18:56 | **与性别同一行**展示；未填写时下游提示词中应说明可按出生日参考、时间不确定 |
| 出生地 | 省下拉 + 市区县输入框/下拉选择 | ✅ | 省份必填，必须从预设省级行政区列表中选择；城市建议必填，县/区可选；省会城市可不填区 | 省份、城市、区县放在同一行展示，便于一次性填写；省份不允许自由输入；示例：北京市、浙江省杭州市、广西壮族自治区南宁市 |
| 性别 | 单选 | ✅ | 男 / 女 / 不便透露 | 用于生成“男命/女命”等表达；不便透露时使用中性表述 |
| 名字中的五行元素 | 复选按钮组 | ❌ | 可多选，最多 5 个；不选表示未提供姓名五行信息 | 金、木、水、火、土；使用明确可见的复选交互；选中后复选框卡片背景变为绿色 |
| 星座 | 下拉选择 | ❌ | 默认根据出生年月日自动计算，允许用户手动修改 | 选项为 12 星座；用于个性化建议，不作为命理主线 |
| MBTI | 下拉选择 | ❌ | 16 型人格 + “不知道/暂不填写” | 用于补充沟通方式、决策偏好和压力应对建议 |
| 当前关注方向 | 复选按钮组 | ❌ | 不限制选择数量 | 工作、感情、财运、学业、考试、健康、人际、家庭；页面文案显示为“当前关注方向”；选中后复选框卡片背景变为绿色 |

### 2.2 大模型职责与下游提示词形态（由 DeepSeek/Kimi 生成）

- **生成方式**：用户点击生成后，后端调用 DeepSeek 或 Kimi，由模型根据用户填写的事实**撰写**最终展示的「优化后提示词」；形态与下方模板**同构**（允许开场句润色，但须包含 **【基础命理信息】**、**【个性化补充信息】** 及 **【提示词生成要求】** 或等价 **【分析任务】** 段落）。
- **职责划分**：本接口返回的正文是「给下游模型用的提示词」；**八字、十神、流年与运势等实质分析**由用户在其它对话中粘贴该正文后，由**下游大模型**执行。本接口调用的模型**不在其回复正文中**对用户输出排盘结果、流年吉凶断语或具体建议（仅撰写下游将遵循的任务说明与用户事实）。

下列 `{占位符}` 仅作结构示意；**实际交付**为模型生成的、已填入真实用户信息的完整文本。

```
你是一个命理学提示词专家，擅长把用户的基础信息整理成可直接提交给 AI 的高质量命理分析提示词。

请根据以下用户信息，生成一段结构化、清晰、专业的命理分析提示词：

【基础命理信息】
- 性别：{gender}
- 出生年月日：{birthDate}
- 出生时间：{birthTime}
- 出生地：{birthPlace}
- 名字中的五行元素：{fiveElements}

【个性化补充信息】
- 星座：{zodiac}
- MBTI：{mbti}
- 当前关注方向：{focusAreas}

【提示词生成要求】
1. 角色设定为命理学专家，同时可参考心理人格分析和星座性格倾向。
2. 八字、十神、流年和逐月运势必须作为分析主线。
3. 星座和 MBTI 只用于补充性格倾向、沟通方式、决策偏好、压力应对方式和建议表达风格，不替代八字分析。
4. 重点围绕用户选择的关注方向展开建议；未选择的方向可简要带过。
5. 生成的提示词需包含近五年运势、今年逐月运势，以及可执行的生活、工作、感情、学习或财务建议。
6. 输出语气应理性、温和，避免绝对化断言，不制造焦虑。
```

### 2.3 内容验证规则

| 验证项 | 规则 | 错误提示 |
|--------|------|----------|
| 出生年月日 | 必填；年份范围 1945 到当前年份；日期必须真实存在；不得超过当前日期 | 请选择正确的出生年月日，出生日期不能晚于今天 |
| 出生时间 | 非必填；填写时支持 HH:mm 或自然语言描述，长度不超过 30 个字符 | 请输入正确的出生时间，或留空 |
| 出生地 | 省份必填且必须为预设选项，城市建议填写，县/区可选 | 请选择正确的出生地 |
| 性别 | 必填；仅允许男、女、不便透露 | 请选择性别 |
| 名字中的五行元素 | 非必填；最多选择 5 个；仅允许金、木、水、火、土 | 请选择正确的五行元素 |
| 星座 | 非必填；必须为 12 星座之一；默认由出生年月日自动计算 | 请选择正确的星座 |
| MBTI | 非必填；必须为 16 型人格之一或“不知道/暂不填写” | 请选择正确的 MBTI 类型 |
| 当前关注方向 | 非必填；不限制选择数量；仅允许预设选项 | 请选择正确的关注方向 |
| 敏感词过滤 | 禁止暴力、色情、政治敏感词 | 输入内容包含敏感词，请重新输入 |

### 2.4 输出展示

- **优化后**：展示 DeepSeek/Kimi 返回的**整段下游用提示词**（即用户应复制到其它对话中的内容）；**不**将本页模型输出展示为「已完成的命理分析报告」。
- **优化前 / 左右对比**：若产品保留对比视图，左侧仅为用户输入摘要，右侧仅为「代写好的下游提示词」；不要求本页大模型生成分析结论。
- **缺失信息**：出生时间、星座或 MBTI 未填时，生成的下游提示词中应说明不确定性或通用处理方式（由上游模型写入下游提示词文本中）。
- **页面对齐**：页头与结果区须用简短文案提示：本页**只生成提示词**，命理解读在粘贴后的下游进行。

### 2.5 交互功能

| 功能 | 说明 |
|------|------|
| 生成优化提示词 | 主按钮文案使用“生成优化提示词”，点击后调用 `/optimize` 生成优化后的提示词，并在结果区域展示；生成中显示加载态；失败时优先展示后端返回的具体校验原因，不只展示通用失败文案 |
| 一键复制 | 保留一键复制功能；复制优化后的提示词到剪贴板，复制成功后按钮文案短暂显示“已复制” |
| 历史记录 | 展示最近若干条「优化后提示词」的预览与时间；不展示用户原始输入；点击可查看完整下游提示词全文 |
| 主题切换 | 支持亮色/暗色主题切换；用户点击按钮后整个页面切换主题，包括页面背景、标题区、表单、卡片、按钮、历史记录和结果区域；暗色模式文字亮度适当降低，避免纯白刺眼 |
| 星座自动计算 | 用户选择出生年月日后自动计算星座，并允许手动修改 |

- **整体风格**：简洁青春风格
- **配色方案**：清新橙红黄色系

### 3.2 主题切换

| 主题 | 适用场景 |
|------|----------|
| 亮色 | 白天使用，清晰明亮 |
| 暗色 | 夜间使用，护眼模式 |

### 4.4 接口设计

#### POST /optimize

说明：

- 后端组装调用 DeepSeek/Kimi 所需的系统侧指令与用户事实；**返回字段 `optimizedPrompt` 为模型生成的正文**，形态与 §2.2 同构，供用户粘贴至下游对话；**不是**后端本地拼接的命理分析结论。
- 调用失败时返回明确错误原因，前端应直接展示该原因。

**请求体：**

```json
{
  "birthDate": "1995-08-18",
  "birthTime": "18:56",
  "birthPlace": {
    "province": "浙江省",
    "city": "杭州市",
    "district": "西湖区"
  },
  "gender": "女",
  "fiveElements": ["木", "水"],
  "zodiac": "狮子座",
  "mbti": "ENFP",
  "focusAreas": ["工作", "感情", "财运"]
}
```


**响应体：**

```json
{
  "id": "history_001",
  "optimizedPrompt": "# 下游系统提示词示例\\n你是一位命理顾问，以下为用户已知信息……\\n（此处为可复制全文，非本接口模型给出的分析结论）",
  "createdAt": "2026-05-08 17:00"
}
```


#### GET /history

**响应体：**

```json
{
  "items": [
    {
      "id": "history_001",
      "createdAt": "2026-05-08 17:00",
      "preview": "# 下游系统提示词示例\\n你是一位命理顾问……"
    }
  ]
}
```

#### GET /history/{id}

**响应体：**

```json
{
  "id": "history_001",
  "createdAt": "2026-05-08 17:00",
  "optimizedPrompt": "# 下游系统提示词示例\\n……"
}
```

前端展示规则：

- 页面首次加载时请求 `/history` 获取最近历史记录列表（条数与展示策略以实现为准；列表可为最新一条或最多 10 条，以实现为准）。
- 列表只展示 `createdAt` 和 `preview`，其中 `createdAt` 使用 `YYYY-MM-DD HH:mm` 格式，`preview` 为优化后提示词前 50 个字。
- 用户每次成功生成提示词后，历史记录列表自动刷新或将新记录插入第一条。
- 点击历史记录时调用 `/history/{id}` 获取完整优化后提示词详情。
- 历史记录展示在结果区域下方，按生成时间倒序排列，不展示用户输入信息。
